1. Initializing git 1.1. git --version 1.2. open the project directory 1.3. git init (one time per project) 1.4. git config --global user.name "JonathanFernz" (one time) 1.5. git config --global user.email jonathanfernzofficial@gmail.com (one time) 2. Add | Commit | Reset | Log | Status | Checkout 2.1. git add .(adding all the changes to the staging area) 2.2. git reset (to unstage changes) 2.3. git commit -m "message" (commits the changes present in the staging area, this step has to be followed after git add) 2.4. git log(viewing all the commits done) 2.5. git status (viewing untracked branches) 2.6. git checkout commit_id(moves the head pointer to a particular commit, changes are reverted temporarily) 2.7. git checkout master(to getback to the changes reverted in previous command) 3. Pushing local repository to github: 3.1. git remote add origin https://github.com/jfernzofficial/GIT-DEMO.git 3.2. git branch -M main 3.3. git push -u origin main 4. Adding new changes to the repository: 4.1. git push origin master OR git push origin main 5. Establishing connection between local terminal and github requires: 5.1. Personal Access tokens 5.2. git push origin main 5.3. Enter github username and personal access token as password
